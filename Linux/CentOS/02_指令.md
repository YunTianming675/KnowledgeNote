## gzip

- 压缩指令，常以 `.gz` 为后缀名，可解压 compress、zip、gzip 等软件的压缩文件
- gzip [选项] 文件名
- 默认会将原来的文件删除，解压缩时同样默认将原文件删除

| 选项 | 文件名                                                       |
| ---- | ------------------------------------------------------------ |
| -c   | 将压缩的数据输出到屏幕上，支持重定向，并保留原来的文件       |
| -d   | 进行解压缩                                                   |
| -t   | 检验文件一致性                                               |
| -v   | 显示原文件/压缩文件的压缩比等信息                            |
| #    | 一个数字，代表压缩等级，-1 最快，但压缩比最差；-9 最慢，但压缩比最好，默认是 -6 |

- 示例

```bash
# 保留原文件
gzip -c services > services.gz
```

## mkisofs

- 创建镜像文件


## tar

- 打包、解包

| 选项        | 含义                                                |
| ----------- | --------------------------------------------------- |
| -c          | 创建打包文件                                        |
| -t          | 查看打包文件的内容有哪些文件名                      |
| -x          | 解包，ctx 不可同时出现在一行命令中                  |
| -z          | 使用 gzip 进行压缩或解压缩                          |
| -j          | 使用 bzip2 进行压缩或解压缩                         |
| -J          | 使用 xz 进行压缩或解压缩，zjJ不能出现在同一行命令中 |
| -f filename | -f 后面要立刻接要被处理的文件名                     |
| -C          | 解包至特定目录                                      |
| -v          | 显示正在处理的文件名                                |
| -p          | 保留原文件的权限和属性                              |
| -P          | 保留文件的绝对路径                                  |
| --exclude   | 排除指定文件                                        |
| --newer     | 仅打包比指定时间新的文件                            |

- 示例

```bash
# 压缩
tar -jcvf filename.tar.bz2 被压缩的文件或目录

# 查询
tar -jtvf filename.tar.bz2

# 解压
tar -jxvf filename.tar.bz2 filename.tar.bz2 -C 欲解压缩的目录

# 仅解压出指定文件
tar -jxvf filename.tar.bz2 待解压的文件
```

- -P 的解释

    不带这个选项时，解压出来的文件就在当前目录下或指定目录下，而带这选项后，解压出来就会覆盖到它的绝对路径上。这个选项容易造成原来的文件被解压后的文件覆盖，但在备份系统时比较有用。

| -u   | 该文件名是否存在且有 SUID 属性       |
| -g   | 该文件名是否存在且有 SGID 属性       |
| -k   | 该文件名是否存在且有 Sticky bit 属性 |
| -s   | 该文件名是否存在且为非空白文件       |

- 两个文件间的比较，如 `test file1 -nt file2` 

| 选项 | 含义                                                         |
| ---- | ------------------------------------------------------------ |
| -nt  | 判断 file1 是否比 file2 新（newer than）                     |
| -ot  | 判断 file1 是否比 file2 旧（older than）                     |
| -ef  | 判断 file1 与 file2 是否为同一文件，可用在 hard link <br>的判定上。主要判定两个文件是否指向同一个 inode |

- 两个整数之间的比较，如 `test n1 -eq n2` 

| 选项 | 含义           |
| ---- | -------------- |
| -eq  | 两数值相等     |
| -ne  | 两数值不等     |
| -gt  | n1 大于 n2     |
| -lt  | n1 小于 n2     |
| -ge  | n1 大于等于 n2 |
| -le  | n1 小于等于 n2 |

- 判定字符串的数据

| 选项              | 含义                                    |
| ----------------- | --------------------------------------- |
| test -z string    | string 是否为 0,是则返回 true           |
| test -n string    | string 是否非 0,是则返回 true,-n 可省略 |
| test str1 == str2 | str1 等于 str2 则返回 true              |
| test str1 != str2 | str1 不等于 str2 则返回 true            |

- 多重条件判定，如 `test -r filename -a -x filename` 

| 选项 | 含义               |
| ---- | ------------------ |
| -a   | 两个状况同时成立   |
| -o   | 两状况任何一个成立 |

# 技巧

1. 查看核心分区记录：`cat /proc/partitions` 
2. 与 CPU 有关的信息：`/proc/cpuinfo` 
3. 关于减号 `-` 的用途：可代表 stdin 或 stdout，如：`tar -cvf - /home | tar -xvf - -C /tme/homeback` 指将 `/home` 目录下的内容打包后传输到 stdout，经过管线后传输给下一个 tar。
